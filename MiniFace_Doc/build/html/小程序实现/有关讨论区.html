5.3.2. 获取讨论列表以及分页查询¶

新建Node.js云函数名称为 getCommunity,建立方式此处不再赘述，代码如下所示：

 1// 云函数入口文件
 2const cloud = require('wx-server-sdk')
 3
 4cloud.init({
 5    env: cloud.DYNAMIC_CURRENT_ENV
 6})
 7const db = cloud.database()
 8const _ = db.command
 9// 云函数入口函数
10exports.main = async (event, context) => {
11    let skip_number = event.skip_number
12    return await db.collection('communityData').orderBy('time_stamp', 'desc').field({
13            '_id': true,
14            '_openid': true,
15            'text': true,
16            'uploader_name': true,
17            'uploader_img': true,
18            'time_stamp':true,
19            'tag_type':true,
20            'img_url_list':true
21        }).skip(skip_number).limit(8)
22        .get()
23        .then(res => {
24            return res
25        })
26        .catch(err => {
27            return err
28        })
29}

communityData 是数据库中的保存讨论信息的集合名称，在云开发控制台中可以自定义，但是不能与其他集合重名。collection() 方法用于获取该集合的引用，返回一个集合的引用。

orderBy() 方法用于对集合进行排序，返回一个排序后的集合的引用。此处是根据时间戳进行降序排序， desc 可以更换成 ascend 以改变排序方式。

field() 方法用于设置返回集合中的字段，当一个数据库中存储字段过多时，可以使用该方法进行过滤，以减少时间开销。

提示

该方法内不能设置false，否则会报错。

limit() 方法用于限制返回集合中的元素个数。

skip() 方法用于跳过集合中的前 n 个元素，通过该方法可以实现分页查询。具体实现方法将在接下来展示。

5.3.3. 页面生命周期函数Onload实现分页-1¶

 1onLoad: function (options) {
 2    let that = this
 3    wx.cloud.callFunction({
 4        name: 'getCommunity',
 5        data: {
 6            skip_number: 0
 7        },
 8        success: function (res) {
 9            // console.log(res);
10            that.setData({
11                community_data: res.result.data
12            })
13            setTimeout(() => {
14                that.setData({
15                    loading_state: false
16                })
17            }, 500);
18        },
19        fail: function (err) {
20            console.log(err);
21        }
22    })
23},

页面加载过程中，调用云函数获取讨论列表，并将数据渲染到页面中。

其中设置的参数 skip_number 默认为0，也就是跳过0个元素，也就是从第一个元素开始返回，返回长度为8.

5.3.4. 页面周期函数OnReachBottom实现分页-2¶

这个生命周期函数是在页面滚动到底部时触发的，但一般页面默认不允许滚动，所以我们要在当前文件同级目录下的json文件中配置允许滚动，如下所示：

1"enablePullDownRefresh": true

设置完成后，开始撰写js代码，如下所示：

 1onReachBottom: function () {
 2    let that = this
 3    wx.showLoading({
 4        title: '加载中',
 5    })
 6    wx.cloud.callFunction({
 7        name: 'getCommunity',
 8        data: {
 9            skip_number: that.data.community_data.length
10        },
11        success: function (res) {
12            // console.log(res);
13            that.setData({
14                community_data: that.data.community_data.concat(res.result.data)
15            })
16            setTimeout(() => {
17                wx.hideLoading()
18            }, 500);
19        },
20        fail: function (err) {
21            console.log(err);
22        }
23    })
24},

通过传入skip_number的值，可以实现分页查询，每次查询8条数据，每次查询的起始位置为上次查询的末尾位置。

提示

优化提示：当页面中数据量较大时，采用上述方式进行页面渲染可能会导致页面开销很大，因为相当于重新刷新了整个界面，会导致页面的掉帧影响用户体验，有一种方式可以对页面数据进行局部调整，有兴趣可自行查阅

5.3.5. 点击事件¶

既然实现了分页查询，接下来面临的问题便是如何点击某个特定的元素从而进入其对应的详细界面呢

仔细看页面的wxml代码会发现， 有绑定一个点击函数 bindtap=”naviToPage”

可以通过 naviToPage 点击事件的回调函数进行查询，代码如下所示：

 1/**
 2 * 页面跳转
 3 * @param {*} e 点击回调
 4 */
 5naviToPage: function (e) {
 6    // console.log(e);
 7    let _id = e.currentTarget.dataset.id
 8    wx.navigateTo({
 9        url: '/pages/pubdetailpage/pubdetailpage?_id=' + _id,
10    })
11},

回看每个元素的数据结构，一个名为 _id 的变量便是区分不同元素的标识，通过回调对象e中的 e.currentTarget.dataset.id 便可以获取当前点击元素的_id， 进而通过url携带参数的形式跳转至详情页面。

在详情页面也就是 pubdetailpage 中,我们便可以根据我们的需求在页面周期函数 Onload 里根据携带的id参数进行数据库的查询，代码如下所示：

 1onLoad: function (options) {
 2    let that=this
 3    wx.showLoading({
 4      title: '正在加载',
 5    })
 6    wx.cloud.database().collection('communityData').doc(options._id)
 7    .get()
 8    .then(res=>{
 9        // console.log(res);
10        that.setData({
11            detail_info:res.data
12        })
13        wx.hideLoading()
14    }).catch(err=>{
15        console.log(err);
16        wx.hideLoading()
17    })
18},

onLoad携带的参数 options ，可以接受上一个页面传过来的id进行查询。
